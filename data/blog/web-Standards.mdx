## 웹 표준성이란

웹 표준성은 World Wide Web Consortium (W3C)와 같은 국제 표준화 기구에서 개발한 웹 기술과 규칙의 집합입니다. 이러한 표준은 HTML, CSS, JavaScript 등의 웹 기술을 포함하며, 웹 개발자들은 이러한 표준을 준수하여 웹 사이트를 구축하고 디자인해야 합니다. 웹 표준을 따르면 웹 사이트의 구조와 콘텐츠가 일관성을 갖게 되어 사용자 경험과 웹 애플리케이션의 성능을 향상시킬 수 있습니다.

## 웹 표준성의 중요성

웹 사이트와 웹 애플리케이션은 표준화된 방식으로 작동하고 구성되어야 합니다. 그 이유는 웹 표준성을 준수하면 웹 사이트가 다양한 브라우저 및 디바이스에서 일관된 방식으로 동작하며, 검색 엔진 최적화 (SEO) 측면에서도 이점을 제공하기 때문입니다.

## 웹 표준성을 지키는 방법

- 적절한 마크업 언어 사용
- 유효한 코드 작성 
- 웹 표준 문서 준수
- 웹 접근성 고려
- 브라우저 호환성 확인

## 웹 표준성과 SEO의 관계

검색 엔진은 웹 사이트의 콘텐츠를 분석하여 색인화합니다. 웹 표준을 준수하면 검색 엔진이 웹 페이지의 구조와 내용을 더 잘 이해할 수 있습니다. 적절한 태그 사용, 시맨틱 마크업, 제목 구조 등은 검색 엔진에게 웹 페이지의 핵심 내용을 전달하는 데 도움이 됩니다.

 - [W3C CSS Validator](http://jigsaw.w3.org/css-validator)
2. [Lighthouse](https://chrome.google.com/webstore/detail/lighthouse/blipmdconlkpinefehnmjammfjpmpbjk)
웹 접근성과 마찬가지로 Lighthouse는 Google에서 제공하는 오픈 소스 웹 개발 도구입니다. 이 도구는 웹 페이지의 품질과 성능을 측정하고 향상시킬 수 있는 가이드를 제공합니다. Lighthouse는 웹 표준 준수 여부, 접근성, SEO, 성능 등 다양한 측면을 검증할 수 있습니다.
